
# ecommerce-growth-analytics-python-sql
A production-style ecommerce growth analytics system using Python and SQL, covering data modeling, A/B experimentation, retention analytics, and business intelligence dashboards.
=======
# E-commerce Growth Analytics Platform

## Project Overview

This project simulates a **complete analytics platform for an e-commerce business**.
It demonstrates how raw customer interaction data can be transformed into actionable insights through a modern analytics stack.

The platform replicates a real-world workflow used by product analytics and data teams, including:

* behavioral data simulation
* ETL pipelines
* data warehousing
* exploratory data analysis
* experimentation analysis
* customer retention analytics
* pricing and margin optimization
* business intelligence dashboards

The objective is to analyze **customer behavior, revenue patterns, and experiment outcomes** to support data-driven decision making.

---

# Business Use Case

Modern e-commerce companies rely heavily on data analytics to answer questions such as:

* What factors influence customer conversion?
* How effective are marketing channels?
* Which product categories drive revenue?
* How does pricing or discounting affect profitability?
* What is the long-term value of customers?

This project simulates an e-commerce platform and builds an analytics pipeline to investigate these questions.

---

# Project Architecture

The platform follows a structured analytics pipeline.

```
Data Simulation
      ↓
Raw CSV Data
      ↓
ETL Pipeline (Python)
      ↓
MySQL Data Warehouse
      ↓
Star Schema Analytics Layer
      ↓
Exploratory Data Analysis
      ↓
Experimentation & Retention Analysis
      ↓
Power BI Dashboard
```

Each stage of this pipeline mirrors common workflows used in modern data teams.

# Data Directory

```
data/
```

This folder stores all datasets generated during the simulation.

```
data/raw/
```

Contains simulated datasets generated by the pipeline.

Files include:

| File            | Description                                       |
| --------------- | ------------------------------------------------- |
| customers.csv   | Customer acquisition channel and experiment group |
| sessions.csv    | User browsing sessions                            |
| cart_events.csv | Cart interaction events                           |

### processed/

```
data/processed/
```

This folder is reserved for transformed datasets created during analysis.

# Source Code Directory

```
src/
```

Contains all core modules responsible for data generation and processing.

---

## config.py

Defines global parameters used across the simulation.

Examples include:

* number of customers
* number of products
* simulation date range
* experiment split ratio
* product category margins
* realistic product price ranges

Centralizing configuration ensures consistency across all modules.

---

## utils.py

Contains reusable helper functions.

Examples:

* saving datasets
* loading datasets
* logging utilities

This prevents duplication of code across multiple modules.

---

## customer_engine.py

Simulates customer profiles.

Responsibilities:

* generate unique customer IDs
* assign acquisition channels
* assign experiment groups (control vs treatment)

This module represents **user acquisition data typically generated by marketing platforms**.

---

## product_engine.py

Generates the product catalog.

The engine assigns:

* product category
* realistic product price
* product cost based on category margin

Price ranges are modeled after **typical Indian e-commerce pricing patterns**.

---

## session_engine.py

Simulates browsing sessions.

Each session contains:

* session duration
* pages viewed
* traffic source
* device type
* bounce indicator

This dataset represents **user behavioral data collected through web analytics tools**.

---

## cart_engine.py

Simulates product interactions with the shopping cart.

Events include:

* product added to cart
* cart abandonment
* cart engagement

These events are critical for **funnel analysis and conversion optimization**.

---

## order_engine.py

Simulates purchases.

Orders are generated based on behavioral signals such as:

* engagement level
* discount sensitivity
* activity propensity

This module creates the **transactional dataset used for revenue analysis**.

---

## metrics_engine.py

Responsible for calculating analytical metrics such as:

* conversion rate
* average order value
* customer lifetime value
* revenue per customer

These metrics support both SQL queries and notebook analyses.

---

## etl_pipeline.py

Handles the **Extract-Transform-Load process**.

Responsibilities include:

1. Extract raw CSV files
2. Transform data types and formats
3. Load datasets into the MySQL database

This module connects Python with the SQL warehouse using SQLAlchemy.

---

# SQL Directory

```
sql/
```

Contains SQL scripts used to structure and query the data warehouse.

---

## schema.sql

Creates the base database schema.

Defines tables for:

* customers
* sessions
* cart events
* orders
* products

These represent the **raw ingestion layer**.

---

## star_schema.sql

Builds the analytics warehouse using a star schema design.

The warehouse contains:

```
dim_customer
dim_product
fact_sessions
fact_orders
fact_cart_events
```

This schema structure is optimized for analytical queries and BI dashboards.

---

## analytics_queries.sql

Contains reusable SQL queries used during analysis.

Examples include:

* conversion rate calculation
* revenue by experiment group
* margin analysis
* customer lifetime value

---

# Notebooks Directory

```
notebooks/
```

Contains analytical investigations performed using Python.

---

## 01_eda_customer_behavior.ipynb

Exploratory analysis of the simulated dataset.

Investigates:

* session duration distribution
* revenue distribution
* product category performance
* customer engagement patterns

---

## 02_conversion_funnel_analysis.ipynb

Analyzes user behavior through the conversion funnel.

Stages analyzed include:

```
sessions → cart interactions → purchases
```

Identifies where the largest drop-offs occur.

---

## 03_ab_experimentation.ipynb

Evaluates the impact of simulated experiments.

Key metrics include:

* conversion lift
* revenue lift
* margin impact
* statistical significance testing

---

## 04_customer_retention_cohort.ipynb

Performs cohort-based retention analysis.

Tracks how customers from different acquisition cohorts behave over time.

Used to estimate **customer lifetime value trends**.

---

## 05_pricing_margin_analysis.ipynb

Examines the relationship between:

* discount levels
* conversion rate
* profit margins

Helps identify pricing strategies that balance growth and profitability.

# How to Run the Project

Follow these steps to run the project from scratch.

---

## 1 Install Dependencies

```
pip install -r requirements.txt
```

---

## 2 Run the Simulation Pipeline

```
python main.py
```

This command will:

* generate simulated datasets
* store raw CSV files
* run the ETL pipeline
* populate the MySQL database
* build the analytics warehouse

---

## 3 Run Notebooks

Launch Jupyter:

```
jupyter notebook
```

Open notebooks in the following order:

```
01_eda_customer_behavior
02_conversion_funnel_analysis
03_ab_experimentation
04_customer_retention_cohort
05_pricing_margin_analysis
```
